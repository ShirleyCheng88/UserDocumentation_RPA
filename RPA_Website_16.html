Eggplant Synergy API


Integrating Eggplant with PathWave Instrument RPA can enhance your automation capabilities and streamline your workflow. The Eggplant Synergy API allows you to easily send requests and interact with RPA through API calls, enabling you to automate your testing process and improve your productivity. In this chapter, we will walk you through the steps and examples to use the Eggplant Synergy API to call PathWave Instrument RPA and perform various automation tasks.

Run RPA API Tests from Eggplant Functional

1. Create a New API Test. In the Eggplant Studio pane, click API Testes to open the API Test Editor. To create a new API test in Eggplant, simply right-click inside the API Tests pane and select New API Test. The new test opens in its own tab in the API Test Editor view in the center pane. You can run and edit API tests in the API Test Editor in Eggplant Functional. See Eggplant API Testing for more details.

API Tests icon in Eggplant Studio Interface

For example, to retrieve the details of Sessions in a Project using the RPA API, you create an API test named "SessionDetails". And then set the HTTP method to "GET" and input the appropriate URL in the address bar. This URL can be obtained from the RPA API Reference documentation. After creating the API test, you can click the green Run button to test whether the API test can be run successfully. In the example provided, we use the following URL:

URL Example: 

                
 http://localhost:3950/api/v1/SessionDetails?projectname=MyProject01
API Test Editor in Eggplant

2. Invoke API tests through Script. Create a SenseTalk Script to call API test. Use the api( ) Function to access all the properties of an API test, including both the request and response.

For example here, we create a script to run the API test we created named “SessionDetails” to display and check the response.

Script Example: 

                

// Make the API call and store the response into RSD object

put SessionDetails() into RSD
                    

// Display the URL in the response object

put RSD.url
                    

// Display the response headers in the response object

put RSD.responseHeaders
                    

// Display the response body in the response object

put RSD.responseBody
                    

// Verify that the URL in the response object contains "MyProject01" using an assertion

assert RSD.url contains "MyProject01"
